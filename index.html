⚠️ Ethical Usage Warning

Do not misuse this tool!

This bot is designed as a safety net or emergency backup for times when you cannot access your laptop (e.g., during travel, illness, or urgent situations) to help you maintain your daily consistency.

It is not intended to automate your entire GitHub presence or falsify your skills. Use it responsibly to bridge occasional gaps, not to replace genuine coding activity.

How to Generate a GitHub Token (PAT)

To authorize this bot to push commits to your repository, you need a GitHub Personal Access Token (classic). Follow these simple steps:

  1. Log in to your account on GitHub.com.
  2. In the top-right corner, click your profile photo and go to Settings.
  3. In the left sidebar, scroll to the bottom and click <> Developer Settings.
  4. Click Personal Access Tokens, then select Tokens (classic).
  5. Click the Generate new token dropdown and select Generate new token (classic).
  6. Give your token a name (e.g., Commit Bot) and set the expiration (e.g., No expiration or 90 days).
  7. Select the scopes: Check the box next to repo (this allows the bot to write to your repositories).
  8. Scroll to the bottom and click the green Generate token button.
  9. Copy the token immediately! (GitHub will never show it to you again). Paste it into the Authentication box in the app.

Are these commits fake?

🔴 HIGHLIGHT: THESE ARE 100% REAL COMMITS!
Every commit made by this tool is a real file update pushed to your GitHub repository. Nothing is simulated or faked. GitHub sees these as valid code contributions, which is why they turn your graph green.

The bot uses the official GitHub REST API to push actual content updates to files in your repository. Because a real commit is created with a unique hash (SHA) on GitHub's servers, they are completely valid and will always be counted in your contributions.

Why does the bot make simple file changes instead of actual code?

You might wonder why the bot modifies files like journal.md or adds hidden comments to README.md instead of writing actual JavaScript or Python code.

1. Safety First: If a bot automatically writes executable code, it could easily introduce syntax errors, break your project's build, or cause your application to crash.

2. Clean Codebase: Automatic code changes would pollute your repository with "junk" code, making it extremely hard to read and maintain.

3. GitHub Graph Rules: GitHub's contribution calendar treats all commits equally. A 1-character comment change counts exactly the same as a 1000-line feature. The bot uses simple, safe updates to guarantee your graph turns green without any risk of breaking your project.

Commit Types Explained

You can choose from three different types of real commits to make your activity look natural:

  • Dev Journal (journal.md): Appends a dated, readable entry to a markdown file in your repository. Keeps a daily log of your consistency.
  • README Update (README.md): Appends a hidden HTML comment containing a timestamp to your README. This updates the file on GitHub but does not change how your README looks to visitors!
  • Code Comment (source files): Appends a standard code comment with a timestamp to a JavaScript file, simulating active code updates.
  • Random Selection (Recommended): For each triggered commit, the app will randomly select one of the three options above. This makes your commit history look organic and varied, rather than modifying the exact same file every day.

Why does 1 commit show as 2 on GitHub?

To protect your streak data, the app automatically creates a streak.json file at the root of your target repository.

Whenever you make a commit:

  1. The app commits your selected file change (e.g., journal.md).
  2. The app immediately makes a second commit to update your streak.json backup.

Since the GitHub API only allows updating **one file per commit**, this results in 2 commits on GitHub. This is actually a great thing because it adds even more activity to your contribution calendar, and both commits are fully recorded!

What is streak.json?

This is a small JSON file stored in your repository that acts as a secure cloud backup for your streak count, last commit date, and date history.

If you clear your browser cache, change browsers, or log in from your phone, the app will read this file from your repository and **instantly restore your streak and calendar graph**!

⚠️ Note on the 30-Day Limit: The calendar grid on this dashboard only shows a rolling window of the last 30 days. Commits older than 30 days will rotate off this visual grid, but they are never lost—they remain permanently saved in your GitHub repository history and your streak.json backup.

How is the Bot Automated?

We provide two ways to automate your daily commits so you never have to worry about forgetting:

🚀 One-Click Cloud Setup (GitHub Actions - Recommended)

Inside the Automation Settings tab, simply select your repository and click One-Click Cloud Setup.

The app will automatically talk to the GitHub API and write the required automation files (.github/workflows/daily-commit.yml and scripts/auto_commit.js) directly into your repository.

GitHub's official cloud servers (GitHub Actions) will then run this script every day at your preferred time for free. This is 100% secure and reliable, working even if your computer and phone are completely turned off!

Option A: Manual Cloud Setup

If you prefer not to use the one-click installer, you can download daily-commit.yml and auto_commit.js manually from the settings tab and upload them to your repository under .github/workflows/ and scripts/.

Option B: Service Worker (Browser Fallback)

If you enable Auto Mode, a Service Worker runs in the background of your browser. If you keep the website tab open (e.g., on a secondary screen or phone browser), the Service Worker will automatically trigger the commit at your preferred time.
